# Phase 4b-Linz: SVF MMSE Plumbing, Mean Word Frequency, and Linz-Style Regression

## Context

This phase has two parts that build on each other:

1. **SVF MMSE plumbing** — mirror the BNT work from Phase 4a for SVF.
   Phase 4a updated the shared data loader to extract MMSE for all three
   tests but only updated the BNT pipeline to consume it. This phase
   updates `svf_pipeline.py` to propagate MMSE into the per-participant
   output CSV, the same way Phase 4a did for `bnt_pipeline.py`. FAS
   plumbing is deferred to a later phase.

2. **Linz-style regression on SVF features against MMSE** — implement
   the SQ4 analysis on SVF data, replicating the methodology of Linz et
   al. (2017): predict MMSE from SVF-derived features (WC, MCS, NOS,
   MWF, SD; MPL skipped because audio is unavailable) using Support
   Vector Regression and Ridge Regression with 5-fold cross-validation,
   reporting MAE with 95% bootstrap CI. The four end-of-day deliverables
   are a correlation table, a regression MAE table, a confusion matrix
   at MMSE = 24 with Cohen's κ, and a predicted-vs-actual scatter
   coloured by diagnosis.

This phase is scoped to **SVF only**. FAS-side Linz analysis and the
multi-model SVF sweep (mirroring Phase 4a's `run_model_comparison.py`
for SVF) are explicitly out of scope and deferred. The SVF cluster
metric pipeline from Phase 3 is upstream of this work and is treated as
fixed; we pin the calibrated threshold (0.45) here and use Phase 3's
existing scorer outputs.

## Critical Methodological Decisions (Pre-Registered)

Three methodological decisions are pre-registered here so they cannot
become post-hoc rationalisations later. Document them in the regression
script's module docstring exactly as stated.

**Threshold for SVF clustering: 0.45.** This is the value calibrated
during Phase 3's threshold sweep. It is the unique threshold at which
both `mean_cluster_size` and `switch_count` cross p < 0.001 in
between-group Kruskal-Wallis testing — the Troyer-style balance point
where clustering and switching contribute roughly equal diagnostic
signal. Pin this value in the SVF config; the regression code does not
re-calibrate.

**Feature selection rule for the regression: |r| > 0.3 with MMSE.**
Mirrors Linz et al. (2017), which excluded MCS from the regression on
the basis of |r| < 0.3 with MMSE in their Table III. We apply the same
rule but to *our* correlation table. Compute Spearman correlations
between every Linz feature (WC, MCS, NOS, MWF, SD) and MMSE; features
with |r| < 0.3 are excluded from the regression. The rule is fixed a
priori; which features actually survive is determined by the data. Both
the full correlation table and the surviving feature subset are
reported.

**Bootstrap CI on MAE: error-based, 1000 iterations.** Run 5-fold CV,
collect held-out predictions for all 100 participants. Bootstrap-resample
the resulting (predicted, actual) pairs 1000 times with replacement;
compute MAE per resample; report 2.5th and 97.5th percentiles as the
95% CI. This is more rigorous than Linz's CV-fold-spread CI, which is
small-N noisy.

### 2. Word frequency module

Create `src/thesis_project/lexical/word_frequency.py` with a pluggable
provider class. The two backends are `wordfreq` (the Python package
used by Linz et al. for direct comparability) and `sucx` (a
SUCX-3.0-derived frequency list, primary Språkbanken-aligned choice).

#### 2a. Public API

```python
from typing import Literal

FrequencySource = Literal["wordfreq", "sucx"]

class WordFrequencyProvider:
    """Pluggable word-frequency lookup with two backends.

    Backend choice:
        - "wordfreq": uses the wordfreq Python package, language='sv'.
          Always available. Used by Linz et al. (2017) for MWF.
        - "sucx": uses a SUCX-3.0-derived frequency list loaded from
          data/external/sucx_frequencies.csv. The file must have
          columns 'lemma' and 'frequency_per_million'. If the file is
          absent, instantiation with source='sucx' raises
          FileNotFoundError with a clear message pointing to the
          preprocessing instructions.

    Frequencies are returned as Zipf scores (log10(frequency_per_billion)
    + 3). This matches the wordfreq package convention and gives a
    stable, interpretable scale across both backends.
    """

    def __init__(
        self,
        source: FrequencySource = "wordfreq",
        sucx_path: Path | None = None,
    ) -> None:
        ...

    def zipf_frequency(self, word: str) -> float:
        """Return Zipf frequency of `word`, lowercased.

        Returns 0.0 for words not found in the underlying resource
        (treated as 'never observed', which on the Zipf scale is the
        floor). This is the wordfreq convention for unknown words.
        """
        ...

    def mean_word_frequency(self, words: list[str]) -> float:
        """Mean Zipf frequency over `words`. Returns NaN for empty input.

        This is the MWF feature in the Linz feature set.
        """
        ...
```

Both backends return Zipf scores so they're directly comparable. The
wordfreq package already produces Zipf scores via its
`zipf_frequency()` function. For the SUCX backend, convert the loaded
frequency-per-million column to Zipf via
`zipf = log10(freq_per_million) + 3` (with a guard against
log10(0) — return 0.0 for absent words).

#### 2b. Robustness for unknown words

Cognitive-impairment responses may include neologisms, paraphasias, or
non-standard forms that don't appear in any frequency list. The MWF
computation must not crash on these. Behaviour:

- Unknown word → Zipf frequency 0.0 (the floor).
- Empty response list → MWF = NaN, with a warning logged once per
  participant.
- Whitespace, punctuation, or empty strings inside the response list
  → filtered out before lookup.

#### 2c. Lemmatisation

Both backends operate on lowercased surface forms by default. Do **not**
add Swedish lemmatisation in this phase; the synthetic data is already
in lemma-like form (single nouns) and adding a lemmatiser introduces a
dependency and a failure mode for marginal gain. Document this as a
limitation in the regression script's docstring.

### 3. `mean_word_frequency` added to SVF scorer

The SVF scorer (whichever module Phase 3 settled on) gains one new
output field: `mean_word_frequency`. It is computed by passing the
participant's valid response list (after deduplication and
preprocessing — i.e. the same list the other consecutive-similarity
metrics operate on) to `WordFrequencyProvider.mean_word_frequency()`.

The frequency source is read from the config (default: `wordfreq`).
Add a `frequency_source` field to `_default_configs.yaml` under the
SVF section:

```yaml
svf:
  cluster_threshold: 0.45      # Calibrated value, Phase 3 sweep
  frequency_source: wordfreq   # 'wordfreq' or 'sucx'; sucx requires
                               # data/external/sucx_frequencies.csv
```

The scorer accepts the provider as an injected dependency rather than
constructing one itself, so tests can pass in a stub. This pattern
keeps the scorer itself frequency-agnostic and testable in isolation.

### 4. SVF Linz regression script

Create `scripts/svf_linz_regression.py`. This is a top-level script,
not a library module. It reads
`data/processed/svf_results_with_mmse.csv` (produced by step 1 above)
and produces four artefacts.

#### 4a. CLI

```
python scripts/svf_linz_regression.py \
    --input data/processed/svf_results_with_mmse.csv \
    --output-dir data/processed/linz/ \
    --figure-dir figures/linz/ \
    --random-seed 42
```

Behaviours:

- `--input` defaults to `data/processed/svf_results_with_mmse.csv`.
- `--output-dir` and `--figure-dir` are created if they don't exist.
- `--random-seed` controls both the CV fold split (StratifiedKFold by
  diagnosis to keep folds balanced) and the bootstrap resampling.
- `--dry-run` prints the planned analysis (model list, feature list,
  CV folds, bootstrap iterations) without running it.

#### 4b. Analysis steps

1. **Load and validate.** Read the input CSV; assert the expected
   columns are present (the 5 Linz features plus `mmse` plus
   `diagnosis`). Drop rows where MMSE is NaN with a logged warning;
   error out if more than 10% of rows are dropped.

2. **Correlation table.** Compute Spearman correlation (use Spearman,
   not Pearson — MMSE has known ceiling effects so Pearson is
   misleading; this is also why Linz's own analysis is fragile, worth
   noting in the docstring) between each of the five Linz features and
   MMSE. Output: `linz_correlation_table.csv` with columns
   `feature, spearman_r, p_value, n, passes_threshold` where
   `passes_threshold` is the boolean `|r| > 0.3`.

3. **Feature selection.** Identify the subset of features with
   `|r| > 0.3`. If the surviving subset is empty, log an error and
   exit with non-zero status (the regression cannot proceed). If
   non-empty, log which features survived and which were dropped, and
   their correlations.

4. **Regression.** For each of two models — Support Vector Regression
   (RBF kernel) and Ridge Regression — fit on the surviving features
   using 5-fold StratifiedKFold (stratified by diagnosis, not MMSE,
   because MMSE is continuous; we want roughly equal diagnostic-group
   representation per fold). Use scikit-learn's `GridSearchCV` for
   inner hyperparameter search:

   - SVR: `C` ∈ {0.1, 1, 10, 100}, `gamma` ∈ {'scale', 'auto', 0.01,
     0.1, 1.0}, RBF kernel.
   - Ridge: `alpha` ∈ {0.01, 0.1, 1.0, 10.0, 100.0}.

   Standardise features (`StandardScaler`) inside each fold to avoid
   leakage. Collect held-out predictions for every participant across
   the 5 folds (each participant predicted once, on the fold where
   they were held out).

5. **Bootstrap MAE.** With the 100 (predicted, actual) pairs collected
   per model, draw 1000 bootstrap resamples (with replacement, size
   100), compute MAE on each resample, return mean MAE and the 2.5th
   and 97.5th percentiles. Output:
   `linz_regression_mae.csv` with columns
   `model, mae_mean, mae_ci_low, mae_ci_high, n_features, features_used`.

6. **Confusion matrix at MMSE = 24.** Convert both true and predicted
   MMSE values to a binary class (≥ 24 = "no impairment", < 24 =
   "possible impairment") and compute the confusion matrix and Cohen's
   κ. Note the cutoff is a clinical convention; document this in the
   script's docstring rather than treating it as data-driven. Output:
   `linz_confusion_matrix.csv` and a separate
   `linz_kappa.csv` reporting κ per model.

7. **Scatter plot.** Predicted vs. actual MMSE, one panel per model
   (so two panels side-by-side), points coloured by diagnosis (HC,
   MCI, non-AD, AD), diagonal reference line, model name and bootstrap
   MAE in the title. Save as
   `figures/linz/svf_predicted_vs_actual.png` at 300 DPI. Use
   matplotlib only; no seaborn dependency.

#### 4c. Logging

The script logs:

- Number of participants loaded after MMSE-NaN filtering.
- Spearman correlations for all 5 features (one line each).
- Surviving features after the |r| > 0.3 filter.
- Best hyperparameters found for each model (one summary line).
- Bootstrap MAE for each model (one line each).
- Cohen's κ for each model.
- Output paths for all artefacts.

All logging at INFO level. The script should be reproducible: a
re-run with the same seed produces byte-identical output CSVs (modulo
floating-point determinism).

```python
def test_wordfreq_backend_returns_zipf_for_known_swedish_word():
    """Common Swedish word ('hund') should have Zipf > 4."""
    provider = WordFrequencyProvider(source="wordfreq")
    assert provider.zipf_frequency("hund") > 4.0


def test_unknown_word_returns_zero():
    """Made-up word returns 0.0, not NaN, not crash."""
    provider = WordFrequencyProvider(source="wordfreq")
    assert provider.zipf_frequency("xkqzj") == 0.0


def test_mwf_handles_empty_list():
    """Empty word list returns NaN, with warning."""
    import math
    provider = WordFrequencyProvider(source="wordfreq")
    assert math.isnan(provider.mean_word_frequency([]))


def test_mwf_filters_whitespace_and_empty():
    """Whitespace and empty strings are filtered before lookup."""
    provider = WordFrequencyProvider(source="wordfreq")
    mwf_clean = provider.mean_word_frequency(["hund", "katt"])
    mwf_dirty = provider.mean_word_frequency(["hund", "", "  ", "katt"])
    assert abs(mwf_clean - mwf_dirty) < 1e-9


def test_sucx_backend_raises_on_missing_file():
    """source='sucx' without the data file gives a clear error."""
    import pytest
    with pytest.raises(FileNotFoundError, match="sucx_frequencies"):
        WordFrequencyProvider(source="sucx", sucx_path=Path("/nonexistent.csv"))


def test_sucx_backend_loads_from_csv(tmp_path):
    """source='sucx' with a fixture CSV returns Zipf frequencies."""
    fixture = tmp_path / "freq.csv"
    fixture.write_text("lemma,frequency_per_million\nhund,500\nkatt,300\n")
    provider = WordFrequencyProvider(source="sucx", sucx_path=fixture)
    # 500/M = 5e-7 = 5e-4 per thousand = log10(5e-4 * 1e9) = log10(500000) ≈ 5.7
    assert 5.0 < provider.zipf_frequency("hund") < 6.0
    assert provider.zipf_frequency("not_in_list") == 0.0
```

## What NOT to Do

- Do **not** modify FAS or BNT pipelines or scorers. SVF only.
- Do **not** modify the data loader. Phase 4a settled it.
- Do **not** re-run the threshold sweep or modify the calibration.
  Threshold is pinned at 0.45 by decision; any deviation is out of
  scope.
- Do **not** add Swedish lemmatisation to the frequency module.
- Do **not** download or generate `sucx_frequencies.csv`. If absent,
  the wordfreq backend is the default and works without it. SUCX
  preprocessing is a separate manual task for the user.
- Do **not** use Pearson correlation with MMSE. Spearman only, because
  MMSE has known ceiling effects and a heavily skewed distribution.
- Do **not** use sklearn's default `cross_val_score` shortcut for
  reporting MAE. We need the held-out predictions themselves (for the
  bootstrap and the scatter plot), so the CV loop must be explicit.
- Do **not** include MCS in the regression by default. Apply the |r| >
  0.3 rule and let the data decide.
- Do **not** report MAE confidence intervals as "fold spread." The
  bootstrap-on-held-out-predictions approach is pre-registered above.
- Do **not** stratify CV folds by MMSE. MMSE is continuous; stratify
  by diagnosis category instead.
- Do **not** save the regression-fitted models to disk. They are not
  the deliverable; the four CSV/figure outputs are.

## Acceptance Criteria

A successful run of this phase produces:

1. All existing tests pass.
2. New tests in all four test files pass under both `pytest -m "not heavy"`
   (fast CI) and `pytest` (full).
3. `python svf_pipeline.py --input data/raw/sweSVFsyntheticData_v3.xlsx`
   produces `data/processed/svf_results_with_mmse.csv` containing
   columns: `participant_id, diagnosis, age, gender, mmse, total_words,
   unique_words, repetitions, mean_consecutive_similarity,
   pairwise_similarity_mean, temporal_gradient, cluster_count,
   mean_cluster_size, switch_count, mean_word_frequency`.
   (Names of Phase-3 cluster metrics may differ; match what the
   scorer actually emits.)
4. `python scripts/svf_linz_regression.py --dry-run` prints the
   planned analysis: 5 features, 2 models, 5-fold CV, 1000 bootstrap
   iterations, 4 output artefacts.
5. `python scripts/svf_linz_regression.py --random-seed 42` runs to
   completion in under 2 minutes on CPU and produces:
   - `data/processed/linz/linz_correlation_table.csv` (5 rows, columns
     `feature, spearman_r, p_value, n, passes_threshold`)
   - `data/processed/linz/linz_regression_mae.csv` (2 rows, one per
     model)
   - `data/processed/linz/linz_confusion_matrix.csv` (one 2×2 matrix
     per model, stacked)
   - `data/processed/linz/linz_kappa.csv` (2 rows, one per model)
   - `figures/linz/svf_predicted_vs_actual.png` (300 DPI, two panels,
     diagnosis-coloured points, diagonal reference)
6. The regression CSV contains MAE values that are sane bounds: MAE ≤
   8 for both models on this synthetic data (MMSE range 0–30; trivial
   "predict the mean" baseline gives MAE ≈ 5–7 depending on
   distribution; a working model should beat that, but MAE > 8 would
   indicate a bug, not just poor performance).
7. Cohen's κ is reported even when it's poor (κ < 0.2). Don't suppress
   negative results.
8. The script exits with a clear error message if no features pass
   the |r| > 0.3 filter, rather than running a regression on an empty
   feature matrix.
